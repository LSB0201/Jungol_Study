# [JUNGOL] 6213 : 학생회장 후보선출

## 📝 문제 설명
$N$명의 학생들이 각각 한 명의 학생회장 후보 $h_i$를 지지하고 있습니다. 교장선생님은 연속된 범위 $[i, j]$의 학생들을 모아 '의견 교환(회의)'을 개최할 수 있습니다. 
* **회의의 규칙:** 모인 학생들 중 **과반수(초과)**가 지지하는 후보가 있다면, 회의에 참여한 **모든 학생이 그 후보를 지지**하도록 마음을 바꿉니다. (과반수 후보가 없다면 아무도 마음을 바꾸지 않습니다.)

교장선생님이 회의를 원하는 만큼 여러 번 개최할 수 있을 때, 최종적으로 **모든 학생이 동시에 같은 후보를 지지하도록 만들 수 있는 가능한 후보들**을 모두 구하는 프로그램을 작성하시오.

---

### 📥 입력 형식
* 첫 번째 줄에는 독립적인 테스트 케이스의 수 $T$가 주어진다. ($1 \le T \le 10$)
* 각 테스트 케이스의 첫 번째 줄에는 학생 수 $N$이 주어진다. ($2 \le N \le 10^5$)
* 두 번째 줄에는 각 학생이 지지하는 후보 $h_i$를 나타내는 $N$개의 정수가 공백으로 구분되어 주어진다. ($1 \le h_i \le N$)
* 모든 테스트 케이스에 대한 $N$의 합은 $2 \times 10^5$를 초과하지 않는다.

### 📤 출력 형식
* 각 테스트 케이스에 대해 한 줄씩 결과를 출력한다.
* 모든 학생들이 동시에 같은 후보를 지지하도록 만들 수 있다면, 가능한 모든 후보 번호를 **오름차순**으로 출력한다.
* 만약 그러한 후보가 단 한 명도 없다면, **$-1$**을 출력한다.
* 숫자를 출력할 때는 공백으로 구분하며, 줄 끝에 불필요한 공백이 없어야 한다.

---

## 💻 입출력 예제

| 입력 예시 (Input) | 출력 예시 (Output) |
| :--- | :--- |
| 5<br>5<br>3 2 2 2 1<br>6<br>4 2 3 4 2 3<br>6<br>6 6 6 2 2 2<br>2<br>2 1<br>3<br>3 2 3 | 2<br>-1<br>2 6<br>-1<br>3 |

## 🛠️ 구조 및 흐름 분석 (학습 가이드)

이 문제는 회의를 여러 번 할 수 있다는 조건 때문에 복잡해 보이지만, **"의견 교환이 단 한 번이라도 성공하여 세력을 확장할 수 있는 최소 단위가 무엇인가?"**를 찾아내면 아주 간단한 문제로 압축됩니다.

### 1. 세력 확장의 핵심 조건 (길이 2 또는 3의 구간)
특정 후보가 과반수를 차지하여 주변 학생들을 포섭하려면, 가장 먼저 **가장 작은 단위에서 과반수**를 달성해야 합니다.

* **길이가 2인 구간 `[X, X]`**: 2명 중 2명이 지지하므로 과반수(1명 초과)를 만족합니다.
* **길이가 3인 구간 `[X, Y, X]`**: 3명 중 2명이 지지하므로 과반수(1.5명 초과 $\rightarrow$ 2명 이상)를 만족합니다.

만약 수열 전체를 통틀어 동일한 숫자가 **바로 이웃해 있거나(`A[i] == A[i+1]`), 하나 건너서 위치(`A[i] == A[i+2]`)하는 경우가 단 한 번도 없다면**, 그 어떤 구간을 잡아도 절대로 초기 과반수를 만들 수 없습니다.

### 2. 단 한 번의 불씨가 있다면 전체를 지배할 수 있는가?
만약 어떤 후보가 `[X, X]` 또는 `[X, Y, X]` 형태로 과반수를 획득해 `[X, X, X]` 형태로 세력을 한 칸이라도 넓히는 데 성공했다면, 그 이후로는 회의 범위를 한 칸씩 옆으로 늘려가며($3$명 중 $2$명, $4$명 중 $3$명...이 지지하는 형태) **전체 $N$명까지 무조건 세력을 확장**할 수 있습니다.

따라서 이 문제는 **"수열 내에 `A[i] == A[i+1]` 또는 `A[i] == A[i+2]`를 만족하는 숫자 `X`가 존재하는가?"**를 찾는 문제로 귀결됩니다.

---

## 🎯 서브태스크별 구현 힌트

* **서브태스크 1 ($N=2$)**: `A[1] == A[2]` 인지 확인하면 됩니다. 같으면 그 숫자 출력, 다르면 `-1`.
* **서브태스크 2 ($N \le 50$)**: 모든 정점 쌍이나 아주 작은 구간을 브루트포스로 완전 탐색해도 통과합니다.
* **서브태스크 3 ($h_i \le h_{i+1}$)**: 수열이 이미 정렬되어 있는 상태입니다. 따라서 같은 숫자는 무조건 이웃해 있으므로 `A[i] == A[i+1]`인 숫자들만 모아서 출력하면 됩니다.
* **서브태스크 4 (추가 제약 없음)**: $O(N)$ 시간 복잡도로 전체 배열을 한 번만 돌면서 `A[i] == A[i+1]`이거나 `A[i] == A[i+2]`인 숫자가 있는지 체크하고, 발견된 숫자들을 `std::set`이나 체크 배열을 통해 중복 없이 모은 뒤 오름차순으로 출력합니다.
