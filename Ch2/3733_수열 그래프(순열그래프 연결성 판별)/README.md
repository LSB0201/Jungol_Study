# [JUNGOL] 3733 : 수열 그래프(순열그래프 연결성 판별)

## 📝 문제 설명
$1$부터 $N$까지의 수열 $A[1], A[2], \dots, A[N]$이 주어질 때, 다음 조건을 만족하는 수들을 같은 그룹으로 묶습니다.
* **조건:** $i < j$ 이고 $A[i] > A[j]$ (즉, 더 앞에 있는 수가 더 큰 경우 = 역전 현상 발생)

수열을 입력받아 위 조건에 맞게 그룹을 나누고, 총 그룹의 개수와 각 그룹의 정보를 규칙에 맞게 출력하는 프로그램을 작성하시오.

---

### 📥 입력 형식
* 첫째 줄에 수열의 길이 $N$이 주어진다. ($1 \le N \le 2,000,000$)
* 둘째 줄에 수열의 각 수가 공백을 구분하여 주어진다.

### 📤 출력 형식
* 첫째 줄에 **총 그룹의 수**를 출력한다.
* 둘째 줄부터 각 그룹의 정보를 한 줄씩 출력한다.
  * 형식: `[그룹에 속한 원소 개수] [원소1] [원소2] ...` (원소들은 **오름차순** 정렬)
* 여러 개의 그룹을 출력할 때는 **각 그룹의 최솟값**을 기준으로 **오름차순** 출력한다.

---

## 💻 입출력 예제

| 입력 예시 (Input) | 출력 예시 (Output) |
| :--- | :--- |
| 6<br>3 1 2 5 6 4 | 2<br>3 1 2 3<br>3 4 5 6 |

---

## 🛠️ 구조 및 흐름 분석 (학습 가이드)

이 문제는 데이터의 크기($N \le 2,000,000$)가 매우 크기 때문에, 단순히 모든 쌍을 비교하는 $O(N^2)$ 알고리즘으로는 시간 초과가 발생합니다. 수열을 순차적으로 탐색하면서 효율적으로 구간을 나누는 선형 시간($O(N)$) 풀이 흐름을 이해해야 합니다.

### 1. 그룹 분할의 핵심 원리
문제의 조건($i < j$ 이고 $A[i] > A[j]$)은 수열에서 **정렬이 역전된 구간**을 의미합니다. 
원소들을 순서대로 확인하면서 **'현재까지 등장한 최댓값'**을 추적합니다. 만약 현재 인덱스 $i$까지 확인했을 때, `현재까지의 최댓값 == i`가 되는 순간이 오면, 그 앞의 원소들은 뒤의 원소들과 절대로 얽힐 일이 없는 하나의 독립된 그룹(컴포넌트)을 완성하게 됩니다.



### 2. 알고리즘 흐름 시뮬레이션 (예제: `3 1 2 5 6 4`)

1. **초기 상태:** `max_val = 0`, 현재 그룹을 담을 임시 공간 준비
2. **i = 1 (값 3):** `max_val = max(0, 3) = 3`. ($3 \neq 1$ 이므로 계속 진행)
3. **i = 2 (값 1):** `max_val = max(3, 1) = 3`. ($3 \neq 2$ 이므로 계속 진행)
4. **i = 3 (값 2):** `max_val = max(3, 2) = 3`. 
   * **[체크]** `max_val == i (3 == 3)` 만족! 
   * 지금까지 거쳐온 원소 `[3, 1, 2]`가 하나의 그룹으로 확정됩니다.
5. **i = 4 (값 5):** `max_val = max(3, 5) = 5`. ($5 \neq 4$ 이므로 계속 진행)
6. **i = 5 (값 6):** `max_val = max(5, 6) = 6`. ($6 \neq 5$ 이므로 계속 진행)
7. **i = 6 (값 4):** `max_val = max(6, 4) = 6`.
   * **[체크]** `max_val == i (6 == 6)` 만족!
   * 새로운 그룹 `[5, 6, 4]`가 확정됩니다.

---

> **💡 힌트/참고 사항**
> * 각 그룹의 원소들을 출력할 때는 오름차순으로 정렬해야 하므로, 그룹이 확정될 때마다 해당 구간의 원소들을 정렬(`std::sort`)하거나, $1$부터 $N$까지의 숫자가 중복 없이 주어지므로 카운팅 정렬 형태를 활용하면 편리합니다.
> * 입출력 데이터의 양이 많으므로 C++의 경우 `ios_base::sync_with_stdio(false); cin.tie(NULL);` 설정을 필수로 추가해야 속도 저하를 막을 수 있습니다.
