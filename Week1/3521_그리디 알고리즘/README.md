# [JUNGOL] 3521 : 튜토리얼 - 그리디(Greedy) 알고리즘

## 💡 개념 설명: 탐욕(Greedy) 알고리즘
1. **정의**: 최적해를 구하는 과정에서 결정을 하는 순간순간마다 가장 최적이라고 생각되는 것을 선택해 나가는 방식으로 최종 해를 얻는 방법입니다.
2. **성립 조건**:
   * **탐욕적인 선택 조건 (Greedy Choice Property)**: 현재 최적이라 선택한 것이 이후의 선택에 영향을 주지 않아야 합니다. (이후에 선택을 번복하는 일이 없음)
   * **최적 부분 구조 (Optimal Substructure)**: 큰 문제의 최적해는 보다 작은 부분 문제의 최적해를 포함합니다.
3. **주의점**: 위 조건을 만족하지 않는 경우 탐욕 알고리즘으로 최적해를 구하기 어렵기 때문에 동적계획법(DP) 등 다른 알고리즘을 고려해야 합니다. 때로는 입력 데이터를 정렬하는 등 전처리가 필요할 수도 있습니다.

---

## 📝 응용 문제 설명
1, 2, 4, 8, 16그램 추가 각각 $a, b, c, d, e$개 주어집니다. 
$N$그램 짜리 물건을 측정하는 데 추의 개수를 최소로 사용하고자 할 때, 사용된 최소 개수의 추를 출력하는 프로그램을 작성하시오.

### 📥 입력 형식
* 첫 번째 줄에 1, 2, 4, 8, 16그램 추의 개수 $a, b, c, d, e$와 목표 무게 $N$이 공백으로 구분되어 입력됩니다.
* 단, $a, b, c, d, e$는 모두 정수이며 모든 추의 개수의 합은 50개 이하입니다. ($1 \le N \le 200$)

### 📤 출력 형식
* $N$그램 짜리 물건을 측정하는 데 사용된 **최소 개수의 추**를 하나의 정수로 출력합니다.
* 만약 주어진 추의 입력으로 해당 무게의 물건을 잴 수 없는 경우 **`impossible`**을 출력합니다.

---

## 💻 입출력 예제

| 입력 예시 (Input) | 출력 예시 (Output) |
| :--- | :--- |
| 10 10 10 0 10 89 | 8 |

> **💡 예제 힌트 (고찰)**
> 무게 89그램을 맞추기 위해 가장 큰 단위인 16그램 추부터 탐욕적으로 최대한 많이 사용해 나갑니다.
> * 16그램 추: 5개 사용 (남은 무게: $89 - 16 \times 5 = 9$그램)
> * 8그램 추: 0개 사용 가능 (입력에서 8그램 추가 0개이므로 건너뜀)
> * 4그램 추: 2개 사용 (남은 무게: $9 - 4 \times 2 = 1$그램)
> * 2그램 추: 0개 사용 (남은 무게 1그램보다 추 단위가 크므로 건너뜀)
> * 1그램 추: 1개 사용 (남은 무게: $1 - 1 \times 1 = 0$그램)
> 
> ∴ 총 사용된 추의 개수: $5 + 0 + 2 + 0 + 1 = 8$개
